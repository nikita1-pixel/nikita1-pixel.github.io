Remote Job Market Analyzer

Where are the remote jobs, and which fit my skills — without refreshing five job boards? A live app that answers it on demand.

🔧 What I built

  • A Streamlit web app that pulls live remote-job postings from a REST API and parses the JSON into a clean table.
  • Interactive filters (search term, category), KPI cards, and Plotly charts that update instantly.
  • One-click CSV export of the filtered results, and deployed live on Streamlit Cloud — shareable by link.

📊 What it shows I can do

  • Consume a live REST API and turn raw JSON into a usable dataset — the everyday reality of real data work.
  • Ship an interactive, filterable dashboard, not a static notebook — analysis anyone can explore themselves.
  • Take a project all the way to deployment — build, host, and share a working product.

✅ Insights for change

  • Turns a manual, repetitive job search into a single self-serve dashboard.
  • The same pattern — API → filter → export — powers any live-data monitoring tool a team needs.
  • Next iterations: cache calls for speed, add more job sources, and score postings against a target skill set.
The difference this makes: it proves I don't just analyse a CSV someone hands me — I can pull live data and ship a working, deployed tool around it.
